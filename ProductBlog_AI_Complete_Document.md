## 1. エグゼクティブサマリー

### 1.1 コンセプト: "Packaging is King"

本サービスの最も革新的な価値提案は、技術的な新奇性ではなく、**「見せ方（パッケージング）」**にあります。
生成AIや自動化ツール自体は既に存在しますが、それらを統合し、「自分の作りたいブログのコンセプトや独自ドメインを入力するだけで、WordPressセットアップから記事のスケジューリング、執筆、投稿までを全自動で運用してくれる環境」をワンストップで提供する点こそが、最大の価値です。

### 1.2 解決する課題

2026年、AIコーディング（Vibe Coding）の普及により、誰もがアプリケーションを開発できるようになりました。しかし、すべての開発者が直面する新たな、そして最大の壁があります。
**「素晴らしいプロダクトを作ったが、誰にも見てもらえない（集客できない）」**という問題です。
開発力はあるがマーケティング力や時間がない個人開発者やスタートアップに対し、「AIツールを使って簡単にブログを生成し、自動で集客し続ける資産（Webサイト）が手に入る」というソリューションを提供します。

## 2. プロジェクト概要

### 2.1 サービス定義

ユーザーのプロダクトやアフィリエイト商材のためのWordPressブログを自動構築・自動運用するSaaS。
簡単なヒアリング（入力）を行うだけで、AIが記事クラスターを提案し、スケジュールに基づいて順次記事を生成・投稿していきます。

### 2.2 コアバリュープロポジション

1.  **認知負荷の最小化**: 「セットアップ3分」のような具体的な時間は目安でしかなく、重要なのは「ユーザーの認知ストレスを極限まで下げる」こと。簡易的な入力だけで資産（ブログ）が積み上がる体験を提供。
2.  **安心感のある出口戦略**: ユーザーがサービス利用をやめたくなった場合でも、構築したWordPressサイトを別のサーバー等へ移管できる「出口（Exit）」を用意。これにより、プラットフォームロックインへの不安を払拭し、導入障壁を下げる。
3.  **資産の蓄積**: SEOの専門知識がない「ライトユーザー」に対し、「運用しているだけでWebページという資産が日々積み上がっていく」というイメージを訴求。

## 3. 市場機会とターゲット

### 3.1 ターゲットオーディエンス

**プライマリーターゲット: 個人開発者・ライトユーザー**

- **属性:** 副業、本業、スタートアップ問わず、全ての個人開発者。
- **インサイト:** ブログ運用は優先度が低い（SNSや動画の方が高い）。しかし、「自動化で埋められるなら、多少費用を払ってでも可能性を買いたい」と考える層。
- **特徴:** SEOの専門家ではない。複雑なSEO施策よりも「自動でやってくれる」簡便さを求めている。

**セカンダリーターゲット:**

- アフィリエイトブログ運営者（プロダクトを持たない初心者含む）
- 既存のブログを拡張したいユーザー

### 3.2 ポジショニング

- **SNS自動化ツールとの違い:** SNS自動化は競争が激しく、細部のクオリティが問われる「レッドオーシャン」。対して、ブログ運用（SEO集客）は開発者にとって優先度が1~2段低いため、そこを自動化することによるポジショニングのズらしが可能。
- **専門性vs大衆性:** ガチガチのSEOマニア向けではなく、「日々自動で資産が積み上がる」という見せ方で、より広範なマス（技術レベルに関わらず、ブログ運用の深い知見がない層）を狙う。

## 4. プロダクト機能要件 (MVP & Future)

### 4.1 機能ロードマップ

| 機能カテゴリ     | MVP要件 (必須)                                                                                              | 将来の拡張性 (Future)                                      |
| :--------------- | :---------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------- |
| **セットアップ** | ・簡易入力のみで完了<br>・サブドメインでのWP設置                                                            | ・独自ドメイン対応<br>・既存ブログへのアドオン機能         |
| **記事生成**     | ・Tavily API等によるセマンティック検索<br>・記事クラスター構想の提案機能                                    | ・より高度なトレンド追従<br>・複数AIモデルの使い分け強化   |
| **運用・投稿**   | ・柔軟なスケジューリング機能<br>・**下書き/自動公開の選択** (設定時必須)<br>・WordPress管理画面での記事編集 | ・アプリUI上での記事編集機能                               |
| **導線設計**     | ・成約用LP/商品へのリンク挿入機能<br>・記事内またはピラーページへの関連付け                                 | ・高度な分析レポート<br>・A/Bテスト自動化                  |
| **インフラ**     | ・弊社管理サーバーでのWP自動構築<br>・バックエンドAPIとWPサーバーの分離                                     | ・他社サーバーへの移管機能（Export）<br>・モバイルアプリ化 |

### 4.2 機能詳細要件

#### 4.2.1 記事クラスター生成とスケジューリング

- **セマンティック検索:** Tavily APIなどを使用し、どのようなブログを作るべきかの記事クラスター（構想）を提案する。
- **スケジューリング機構 (MVPの核):** 「毎日1記事」といった固定ではなく、ユーザーが指定したスケジュール（例：1日10記事、1日1回3記事など）に基づいて生成ジョブを走らせる柔軟な設計。手動生成のみのMVPも検討範囲だが、スケジューリングは重要視。
- **チェック機能:** 設定段階で記事を「下書き保存」するか「自動公開」するかチェック項目を設ける。
  - ユーザーによる記事チェックは、基本的にアプリ上ではなく、生成されたWordPressの管理画面（下書き一覧）で行うフローとする。

#### 4.2.2 コンバージョン（成約）への導線

- ユーザーが成約させたいLPや商品（アフィリエイトリンク等）を設定可能にする。
- 記事生成時、文末や文中、ピラーページへのリンクとして、自然な形で成約ポイントへの誘導経路を埋め込む。

#### 4.2.3 WordPress環境

- **テーマ・プラグイン:** 基本的に弊社側で用意したデフォルトテーマ・プラグインを初期セットアップする。
- **ユーザー権限:** ユーザーはWordPressの管理者権限を持ち、WP管理画面から自由にテーマ変更やプラグイン追加、記事の手動修正が可能。
- **サーバー分離:** バックエンド（APIビルド）サーバーとWordPress設置サーバーは明確に分離し、セキュリティとSEO対策（表示速度等）を担保する。

## 5. 技術アーキテクチャ

### 5.1 基本構成

- **CMS:** WordPress (必須確定)
- **サーバー:** DigitalOcean (最優先候補)
  - 将来的なスケーラビリティ（500〜1000ユーザー以上）を考慮しつつ、初期はコストを抑える。
- **AIモデル (LLM):** ソフトコーディングにより複数モデル選択可能にする。
  - デフォルト想定: **Gemini** (暫定)
  - 画像生成: **NanoBananaPro** (暫定・ソフトコーディング)
- **検索API:** Tavily API (セマンティック検索・情報収集)

### 5.2 セキュリティ要件 (最優先事項)

全てのセキュリティリスクを網羅的に排除する設計とする。

- WAF (Web Application Firewall) の導入（Cloudflare）
- DDoS攻撃対策（Cloudflare）
- XSS (Cross-Site Scripting) 対策
- SQLインジェクション対策
- ユーザーデータの厳格な保護
- WordPress脆弱性対策（プラグイン最小化、定期更新）
- Nginxセキュリティヘッダー設定

### 5.3 モバイル対応

将来的なモバイルアプリ化、またはモバイルブラウザでの利用率高騰を見越し、対応可能な技術スタックで構築する（レスポンシブデザインの徹底、APIファーストな設計）。

## 8. リスク分析と対策

### 8.1 プラットフォームロックインへの不安

- **リスク:** ユーザーは「サービスが終了したらブログも消えるのでは？」「使いにくい時にやめられないのでは？」という不安を持つ。
- **対策 (Exit Strategy):** 「ツールで作成したサイトや記事データを、他のサーバーへそのまま移管できる」ことを明示し、いつでも逃げられる出口を用意することで、登録・利用の心理的ハードルを下げる。

### 8.2 SEOの質と評価

- **リスク:** 完全自動生成記事がGoogleに低評価を受ける（スパム扱い）。
- **対策:**
  - Tavily APIを用いたセマンティック検索で「意味のある」記事構成を提案。
  - 単なるキーワード埋め込みではなく、記事クラスター構造によるサイト全体の専門性向上。
  - ユーザーによる手動チェック（下書き保存機能）の推奨。
