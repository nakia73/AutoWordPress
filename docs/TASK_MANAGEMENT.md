## プロジェクトについて

このリポジトリは **Argo Note** のプロダクト仕様を定義するためのドキュメントプロジェクトです。

**重要:** このプロジェクトは**仕様定義**が目的であり、**実装**は含みません。

- コードの実装は別リポジトリで行う
- ここでは設計・仕様・決定事項を管理
- ドキュメントの完成度を高めることが目標

---

## デプロイメント環境

### 現在のリリース環境

| 項目 | 詳細 |
|------|------|
| **アプリケーション形態** | **ウェブアプリケーション（SaaS）** |
| フロントエンド | Next.js 14+ （Vercel ホスティング） |
| バックエンド | Next.js API Routes + WordPress Multisite（DigitalOcean VPS） |
| 対象デバイス | PC / タブレット / スマートフォン（レスポンシブ対応） |
| 対象ブラウザ | Chrome, Safari, Firefox, Edge（最新2バージョン） |

### 将来のプラットフォーム展開計画

| Phase | プラットフォーム | 方針 |
|-------|------------------|------|
| MVP〜Phase 15 | ウェブアプリ | 現在の主要開発対象 |
| Phase 16（予定） | モバイルアプリ | ウェブアプリをPWA or ネイティブラッパーでモバイル対応 |

**モバイルアプリ対応方針:**
- まずウェブアプリとして完成させる（Phase 6: MVP Launch）
- ユーザーフィードバックに基づきモバイル需要を評価
- 需要が確認された場合、以下のいずれかでモバイル対応：
  1. **PWA（Progressive Web App）**: 最小工数でのモバイル対応
  2. **React Native / Expo**: iOS/Android両対応ネイティブアプリ
  3. **Capacitor**: 既存Next.jsコードを活用したハイブリッドアプリ

## タスク登録ガイド

### 新規タスクの登録方法

1. 「未着手」セクションに行を追加
2. 以下の情報を記入：
   - **ID:** T + 連番（例: T008）
   - **タスク:** 簡潔な説明（50文字以内）
   - **優先度:** 高 / 中 / 低
   - **依頼日:** YYYY-MM-DD
   - **備考:** 追加情報（任意）

### タスクのステータス変更

- 着手時: 「未着手」→「進行中」に移動
- 完了時: 「進行中」→「完了タスク」に移動、成果物を記載

### 優先度の基準

| 優先度 | 基準 |
|--------|------|
| 高 | MVP仕様に直接影響 / ブロッカー |
| 中 | 仕様の明確化に必要 |
| 低 | あれば良い / 将来対応可 |
