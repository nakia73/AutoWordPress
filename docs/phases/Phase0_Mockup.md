## 重要: Phase 0 と Phase 0.5 の役割分担

| 項目 | Phase 0（本フェーズ） | Phase 0.5（次フェーズ） |
|------|----------------------|------------------------|
| 目的 | コンセプト検証 | 公開用ブランド作成 |
| 成果物 | モックアップ動画、スクリーンショット | ロゴ、アイコン、LP |
| ブランディング | プレースホルダーでOK | 90%+品質が必要 |
| 公開範囲 | SNSティーザー | 正式LP（ウェイトリスト収集） |
| コスト | 最小限（仮デザインで可） | 必要な投資を行う |

**流れ:**
```
Phase 0: モックアップ作成・SNS反応テスト
   ↓ （反応が良ければ）
Phase 0.5: ブランドアセット作成・正式LP公開
   ↓
Phase 1〜: 本開発開始
```

## 1. 目的 (Purpose)

「実際に作る」前に「欲しい人がいるか」を確認します。特に X (Twitter) や Threads で話題になるような、**「URLを入れただけで魔法のようにブログができる」瞬間を演出すること**が最大の目的です。

**ビルドインパブリック / プロセスエコノミー**として需要をテストしながら開発を進めます。

> **詳細:** [CONCEPT_DECISIONS.md - B11. ビルドインパブリック戦略](../CONCEPT_DECISIONS.md#b11-ビルドインパブリック戦略需要検証コミュニティ形成)

## 2. 成果物

### 2.1 デモ動画・スクリーンショット（SNS投稿用）

**目的:** SNSでの反応を測るためのビジュアル素材

- **訴求:** 「Your AI-Powered Blog. Fully Automated.」
- **形式:** 30秒〜1分のデモ動画、スクリーンショット
- **ブランディング:** プレースホルダー（仮ロゴ、仮アイコン）でOK

**注意:** この段階では正式なLPは作成しない。正式LPは [Phase 0.5](./Phase0.5_MVPBranding.md) でブランドアセット作成後に公開。

### 2.2 セットアップ演出 (The "Magic" Moment)

- 「分析中...」「キーワード抽出中...」「WordPress構築中...」といった、裏側で複雑な処理が走っていることを期待させるリッチなアニメーション演出。
- **これはモックアップ動画用の素材**として作成

### 2.3 管理画面シミュレーション

- 記事が自動で生成され、WordPress（風の画面）に投稿されていく様子のモックアップ。
- **静止画またはアニメーションGIF**として作成し、SNS投稿に使用

## 3. マーケティング活動

### Phase 0 での活動（本フェーズ）

- **SNS展開:** モックアップ動画をX や Threads で「開発中のプロダクト」として投稿
- **反応測定:** いいね、RT、コメント数を観測
- **目的:** コンセプトへの興味・関心を確認

### Phase 0.5 での活動（次フェーズ）

- **正式LP公開:** ブランドアセット完成後にウェイトリスト収集開始
- **検証:** 50名以上のウェイトリスト登録を目指し、市場の需要を定量的に測定

## 4. ビルドインパブリック戦略の実行

### 4.1 開発哲学

本プロジェクトは「**先行投資型アプローチ**」を採用している。

> 将来的にAIブログ運用ツールが必須ツール化される市場に**先行参入**するため、
> 現時点での需要・効果検証に固執せず、市場成熟前に地盤を固めておく戦略。

**詳細:** [CONCEPT_DECISIONS.md - B10. 先行投資型アプローチ](../CONCEPT_DECISIONS.md#b10-先行投資型アプローチの戦略的根拠)

### 4.2 プロセスエコノミーの実践

> 完成品を売るのではなく、**作る過程自体を価値に変える**。

| アクション | 頻度 | 目的 |
|-----------|------|------|
| 開発進捗の発信 | 毎日 | 透明性・信頼構築 |
| 設計判断の共有 | 随時 | ユーザーの巻き込み |
| 失敗・試行錯誤の公開 | 随時 | 共感・親近感の醸成 |
| フィードバック募集 | 週1回 | プロダクト改善 |

### 4.3 需要検証フロー

```
【Phase 0】
Step 1: コンセプト発信
   │  - 「こんなツールがあったら使いたいか？」を問いかけ
   ▼
Step 2: モックアップ・デモ動画公開
   │  - 反応（いいね、RT、コメント）を定量観測
   ▼
Step 3: 初期反応の評価
   │  - 「反応が良ければ次へ」「反応が悪ければピボット検討」
   ▼
【Phase 0.5】
Step 4: ブランドアセット作成・正式LP公開
   │  - 90%+品質のロゴ・アイコン
   │  - ウェイトリスト収集開始（目標: 50名）
   ▼
Step 5: 登録者とのディスカッション
   │  - 「何に最も価値を感じるか」「何が不安か」をヒアリング
   ▼
Step 6: 軌道修正
   │  - フィードバックを基に機能優先度・訴求軸を調整
   ▼
【Phase 1〜】
Step 7: 本開発開始
```

### 4.4 発信チャネル

| チャネル | 頻度 | 内容 |
|---------|------|------|
| X（Twitter） | 毎日 | 開発進捗、学び、小ネタ |
| Threads | 毎日 | Xと連動 |
| note/Zenn | 週1-2回 | 技術的な深掘り記事 |
| Product Hunt | リリース時 | ローンチ告知 |
| Indie Hackers | 週1回 | 進捗レポート、議論参加 |

### 4.5 軌道修正の判断基準

| 指標 | 良好 | 要検討 | ピボット検討 |
|------|------|--------|-------------|
| ウェイトリスト登録（2週間） | 50+ | 20-49 | <20 |
| モックアップ動画の反応率 | 5%+ | 2-5% | <2% |
| 「お金を払ってでも使いたい」の声 | あり | 曖昧 | なし |

## 5. 成功基準

### Phase 0（本フェーズ）の成功基準

- モックアップ動画がSNSで一定の反応（いいね50+、RT10+など）を得られること
- 「これ欲しい」「使いたい」というコメントが複数得られること
- コンセプトへの明確な否定的反応がないこと

### Phase 0 + 0.5 を通じた成功基準

- 50人以上のウェイトリスト登録（Phase 0.5 完了後）
- ユーザーからの具体的なフィードバックを3件以上収集
- 「お金を払ってでも使いたい」という強い需要シグナルの確認

---

## 6. 「大きなニーズを掘り当てる」マインドセット

> 最初から完璧なプロダクトを作ろうとしない。
> ユーザーとのコミュニケーションの中で、**細かく軌道修正しながら、大きなニーズを掘り当てていく**。

- **初期仮説:** 「個人開発者はAI自動ブログを求めている」
- **検証方法:** ビルドインパブリックでの反応観測
- **発見:** 実際のニーズは仮説と異なる可能性がある
- **対応:** ニーズに合わせてピボット or 機能調整

これは**リーンスタートアップ**の考え方と整合する：
- Build（構築）→ Measure（計測）→ Learn（学習）のサイクル
- 最小限のコストで市場検証
- 失敗を早く・安く行い、成功パターンを見つける
