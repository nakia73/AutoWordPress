# Phase 13: Brand Evolution - ブランド進化・リブランディング

> **サービス名:** Argo Note（※本Phaseで再検討の可能性あり）
> **関連ドキュメント:** [開発ロードマップ](../DEVELOPMENT_ROADMAP.md) | [コンセプト決定](../CONCEPT_DECISIONS.md) | [Phase 0.5: MVP Branding](./Phase0.5_MVPBranding.md)
> **優先度:** 低（成長フェーズ後半）
> **前提:** MVP運用が安定し、ユーザーフィードバックが蓄積された段階

## 概要

[Phase 0.5](./Phase0.5_MVPBranding.md) で作成した MVP Branding を進化させ、ユーザーフィードバックを反映した完成度の高いブランドアイデンティティを確立するフェーズです。

### Phase 0.5 と Phase 13 の関係

| 観点 | Phase 0.5（MVP Branding） | Phase 13（Brand Evolution） |
|------|---------------------------|----------------------------|
| タイミング | MVP開発前（Week 1） | 成長フェーズ |
| 目的 | LP公開に必要な最小限 | 完成度の高いブランド確立 |
| 品質水準 | 90%（十分良い） | 100%（記憶に残る） |
| 根拠 | 仮説ベース | ユーザーフィードバックベース |
| 成果物 | ロゴ、アイコン、カラー | ガイドライン、全アセット、グッズ |

**なぜこのタイミングか:**
- MVP運用でプロダクトの方向性が確定している
- 実ユーザーのフィードバックを反映したブランディングが可能
- 成長フェーズで認知拡大する前に「記憶に残る」アイデンティティを確立
- Phase 14（リファラルプログラム）のグッズ制作と連携

## 実施項目

### 1. プロダクト名の再評価

**前提:** Phase 0.5 で決定した名前（Argo Note）での運用実績がある

**検討事項:**
- 現在の名前でユーザーに認知されているか
- ネガティブなフィードバックはないか
- リブランディングのメリット・デメリット
- グローバル展開での問題はないか

**判断基準:**
- 「変更すべき明確な理由」がない限り、継続を推奨
- リブランディングコストは高い（既存認知の喪失）

**成果物:**
- [ ] プロダクト名の継続/変更の最終決定
- [ ] 変更の場合: 新ドメイン取得、移行計画
- [ ] 商標登録（成長に伴い必要であれば）

---

### 2. ブランドコンセプト・ストーリー設計

**検討事項:**
- ブランドの核となるメッセージ
- ブランドパーソナリティ（どんな人格か）
- トーン&マナー（どんな話し方をするか）
- ブランドストーリー（なぜ生まれたか）

**参照:** CONCEPT_DECISIONS.md A5（原体験）、A6（5年後のビジョン）

**成果物:**
- [ ] ブランドガイドライン（コンセプト編）
- [ ] タグライン/キャッチコピーの確定

---

### 3. アイコン・ロゴの洗練

**前提:** Phase 0.5 で作成したロゴ・アイコンが運用されている

**評価事項:**
- 現在のロゴへのユーザー反応
- 競合との差別化度合い
- 様々な用途（SNS、グッズ、広告）での適用性
- 改善すべき点の洗い出し

**進化オプション:**
1. **維持:** 現状のまま継続（問題がなければ推奨）
2. **微調整:** 細部の改善（色味、バランス等）
3. **リデザイン:** 大幅な変更（問題がある場合のみ）

**成果物:**
- [ ] ロゴ（完成版: SVG、PNG各サイズ）
- [ ] アイコン（全プラットフォーム対応版）
- [ ] ロゴ使用ガイドライン（詳細版）
- [ ] グッズ用デザインバリエーション（Phase 14連携）

---

### 4. ビジュアルアイデンティティ

**検討事項:**
- プライマリカラー、セカンダリカラー
- タイポグラフィ（フォントファミリー）
- アイコンスタイル
- 写真・イラストのトーン
- UIコンポーネントのスタイル

**成果物:**
- [ ] カラーパレット定義
- [ ] タイポグラフィガイド
- [ ] ビジュアルスタイルガイド

- [ ] App Store / Play Store 用アイコン（将来用）

## 外部リソース検討

| 選択肢 | メリット | デメリット |
|--------|---------|-----------|
| 自作 | コスト最小 | クオリティに限界 |
| デザイナー委託 | プロクオリティ | コスト発生 |
| AIツール活用 | 低コスト・高速 | 独自性に欠ける可能性 |
| コンペ形式 | 複数案から選択可能 | 管理コスト |

**推奨:** ロゴ・アイコンは外部デザイナーへの委託を検討

---

## 成功指標

- ブランドガイドラインの完成
- 全アセットの制作完了
- プロダクト全体へのブランド適用完了
- ユーザーからの「印象に残る」フィードバック

---

## 注意事項

- Phase 0.5 のブランディングは「暫定版」ではなく「初版」として扱う（十分な品質がある）
- 変更は「明確な改善理由」がある場合のみ実施（変更コストを考慮）
- ユーザーフィードバックを反映したブランディングを心がける
- グローバル展開を前提とした設計（多言語対応、文化的配慮）
- Phase 14（リファラルプログラム）のグッズ制作を見据えた設計
