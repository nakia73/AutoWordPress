## なぜこのフェーズが必要か

### Phase 0 と Phase 0.5 の違い

| 観点 | Phase 0（Mockup） | Phase 0.5（MVP Branding） |
|------|-------------------|---------------------------|
| 目的 | コンセプト検証・デモ作成 | 公開用ブランドアセット作成 |
| アウトプット | 動画、スクリーンショット | ロゴ、アイコン、カラーパレット |
| 公開範囲 | SNS投稿（ティーザー的） | 正式LP、プロダクト全体 |
| 品質水準 | プレースホルダーでOK | 90%+の完成度が必要 |
| コスト | 最小限 | 必要な投資を行う |

## 成果物

### 1. ロゴ・アイコン（必須）

| アセット | 用途 | 形式 |
|---------|------|------|
| ロゴ（フルカラー） | LP、Webサイト | SVG, PNG |
| ロゴ（モノクロ） | 印刷、暗い背景用 | SVG, PNG |
| アイコン（正方形） | ファビコン、SNS | PNG 512x512 |
| アイコン（小） | ファビコン | ICO, PNG 32x32 |

**品質基準:**
- プロフェッショナルな印象を与える
- 視認性が高い（小サイズでも認識可能）
- プロダクトの性質を反映している

### 2. カラーパレット（必須）

| 役割 | 用途 |
|------|------|
| プライマリカラー | CTAボタン、アクセント |
| セカンダリカラー | 補助的な強調 |
| 背景色 | ページ背景 |
| テキスト色 | 本文テキスト |

### 3. タイポグラフィ（必須）

- ヘッドライン用フォント
- 本文用フォント
- （Google Fonts または システムフォントを使用）

### 4. OGP画像（必須）

- SNSシェア時に表示される画像
- 1200x630px（推奨）

## 制作アプローチ

### 選択肢の比較

| アプローチ | コスト | 時間 | 品質 | 推奨度 |
|-----------|--------|------|------|--------|
| 自作（Figma等） | 無料 | 2-3日 | 中〜高 | デザインスキルがあれば○ |
| AIツール活用 | 無料〜低 | 1日 | 中 | 初期案として○ |
| デザイナー委託 | $100-300 | 3-5日 | 高 | 予算があれば◎ |
| ロゴジェネレーター | $20-50 | 即日 | 中 | 迅速に必要な場合○ |

### 推奨フロー

```
1. コンセプト言語化（1時間）
   │  - プロダクトの性格
   │  - 伝えたい印象
   │  - 避けたい印象
   ▼
2. 参考収集（2時間）
   │  - 競合のロゴ分析
   │  - 好みのデザイン収集
   ▼
3. ラフ案作成（半日）
   │  - 複数の方向性を試す
   │  - AIツールも活用
   ▼
4. 精緻化（半日〜1日）
   │  - 選んだ方向性を磨く
   │  - 各サイズでの視認性確認
   ▼
5. アセット書き出し（2時間）
   │  - 必要な形式・サイズで出力
   ▼
```

## 品質チェックリスト

### ロゴ・アイコン
- [ ] 16x16pxで認識できる
- [ ] 白背景、暗い背景の両方で使える
- [ ] 競合と差別化されている
- [ ] プロダクトの価値を反映している

### カラーパレット
- [ ] アクセシビリティ基準を満たす（コントラスト比）
- [ ] 一貫した印象を与える
- [ ] Web Safe Color である必要はないが、画面での表示を確認済み

### 全体
- [ ] 「プロフェッショナル」な印象を与える
- [ ] 「信頼できる」サービスに見える
- [ ] ターゲット（バイブコーダー）に響く

---

## Phase 13 への引き継ぎ事項

MVP Branding で作成したアセットは「暫定版」として扱う。
Phase 13 で以下を実施：

- ユーザーフィードバックを反映したリブランディング検討
- 完全なブランドガイドライン策定
- 追加アセット（プレゼン資料、メールテンプレート等）
- グッズ用デザイン（Phase 14 リファラルプログラム連携）

---

## 成功基準

- LP公開に必要なすべてのアセットが揃っている
- ターゲットユーザーから「プロフェッショナルに見える」フィードバックを得られる
- Phase 1 以降の開発でブランドの一貫性が維持できる

---

## 注意事項

- **完璧を求めすぎない:** 90%の品質で前に進む。100%は Phase 13 で。
- **速度重視:** LP公開を遅らせないことが最優先
- **イテレーション前提:** 最初から完璧である必要はない
- **コスト意識:** 月額予算$100の制約を忘れない
