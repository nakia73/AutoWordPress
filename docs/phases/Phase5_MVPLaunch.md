# Phase 5: MVP Launch（リリース）詳細仕様書

**テーマ:** Real World Validation
**ゴール:** 最初の10社のユーザーにサービスを実際に提供し、実益を生むプロダクトであることを証明する。

内部テストを終え、市場の洗礼を受けるフェーズです。ここでの最大の目的は売上ではなく、**「熱狂的なフィードバック」**と**「実際にSEO効果が出ることの初期確認」**です。

## 3. 実装 & 運用ステップ

### Step 1: WordPressテーマの選定と最適化

- 無料テーマ（例: GeneratePress の無料版など）の中から、検証結果が「SEO最適、高速、モバイルフレンドリー」であるものを1つ選定。
- ユーザーが個別にテーマを選ぶ手間を省き、最初から「勝てる設定」をプリセットとして配布。

**テーマ選定基準:**

| 基準 | 要件 |
|------|------|
| PageSpeed | モバイル90点以上 |
| Core Web Vitals | LCP < 2.5s, FID < 100ms, CLS < 0.1 |
| Schema対応 | Article, FAQ, Breadcrumbに対応 |
| 多言語対応 | 日本語表示が崩れない |

### Step 2: 10社限定招待 & 集中サポート

- Phase 0で収集したウェイトリストの中から、フィードバック意欲の高い10社を選定し個別招待。
- Slackチャンネルを開設し、開発者がダイレクトにユーザーの困りごとを解決する体制。

**招待基準:**

| 優先度 | 条件 |
|--------|------|
| 高 | SaaSプロダクトを運営している |
| 高 | フィードバックに積極的 |
| 中 | SEOに課題を感じている |
| 中 | コンテンツマーケに興味がある |

### Step 3: フィードバックループの構築

- 週次の利用状況アンケートと、NPS（継続意向）の計測。
- ユーザーの言葉をその場でバックログ（Phase 6以降の計画）に反映。

**収集する指標:**

| 指標 | 測定方法 | 目標値 |
|------|----------|--------|
| NPS | 週次アンケート | 30以上 |
| 記事生成数 | 管理画面から集計 | 平均5記事/週 |
| ログイン頻度 | 分析ツール | 週3回以上 |
| 問い合わせ数 | Slack/メール | 減少傾向 |

### Step 4: 成功事例の収集

- インデックス登録の進捗、PVの初動などをウォッチ。
- 「開始1週間でGoogleに載った」という初動の成功を積極的にシェアする。

## 4. 監視・運用体制

### リアルタイム監視


|------|------|----------|
| データベース | 毎日 | 7日間 |
| アップロードファイル | 毎日 | 7日間 |
| 設定ファイル | 変更時 | 30日間 |

---

## 5. セキュリティ & 実用性の考慮

- **リソース監視:** 急激な負荷増大に備え、VPSのメトリクス（CPU/メモリ）を常時監視。
- **バックアップ:** 万が一のトラブルに備え、DBとMultisite全体のデイリーバックアップを自動化。
- **インシデント対応:** 重大な問題発生時の連絡フローを事前に策定。
